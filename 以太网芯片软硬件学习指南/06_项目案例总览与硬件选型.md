# 06 项目案例总览与硬件选型

## 本章解决什么问题

前面章节分别讲了 PHY、Switch、NIC 和联调方法。本章把它们放进真实项目里看：一个以太网产品从需求开始，如何选择芯片、接口、软件架构和验证路径。

真实项目通常不是“只会一个协议”就能做出来，而是下面这些环节一起成立：

```text
应用需求
  |
速率、端口数、延迟、可靠性、成本
  |
芯片选型：PHY / Switch / NIC / FPGA / SoC
  |
硬件设计：电源、时钟、reset、接口、电平、PCB
  |
底层 bring-up：ID、link、速率、计数器
  |
驱动或 Firmware：寄存器、DMA、队列、中断
  |
协议栈：ARP/IP/ICMP/UDP/TCP/VLAN/Modbus
  |
应用验证：ping、iperf、echo、业务协议
  |
现场排障：抓包、日志、统计、示波器、ILA
```

## 五类代表项目

| 项目 | 典型硬件 | 软件形态 | 学习重点 |
| --- | --- | --- | --- |
| ZYNQ 千兆 PHY | RJ45 PHY + RGMII + PS GEM | Vitis 裸机 lwIP | PHY、MDIO、GEM、DMA、pbuf、TCP echo |
| 嵌入式 Switch 网关 | Switch ASIC + CPU Port + 多 PHY | Linux DSA 或厂商 SDK | VLAN、PVID、MAC 表、CPU trap、端口统计 |
| PCIe NIC 网卡 | PCIe NIC + MAC/PHY/SerDes | Linux 网卡驱动 | BAR、MMIO、DMA ring、NAPI、offload |
| FPGA 10G UDP 采集 | SFP+ + GT + 10G MAC + DMA | RTL + Firmware/上位机 | AXI-Stream、PCS/PMA、UDP、高吞吐 |
| 工业以太网控制 | MCU/SoC + PHY/Switch | TCP socket 或 lwIP | 静态 IP、Modbus TCP、重连、现场排障 |

这些项目不是互相孤立的。它们共享同一条底层逻辑：

```text
先让物理链路可靠，再让以太网帧正确，再让协议栈工作，最后才验证应用。
```

## 芯片选择的核心问题

### PHY 选择

先问：

```text
需要 10/100M、1000M、2.5G、10G，还是多速率？
接口接 MAC 用 RMII、RGMII、SGMII、USXGMII 还是 10GBASE-R？
是铜口 RJ45，还是光口 SFP/SFP+？
MAC 侧 IO 电压和 PHY IOVDD 是否匹配？
是否需要 IEEE 1588 PTP 硬件时间戳？
是否需要 EEE、Wake-on-LAN、工业温度、车规等级？
```

代表学习锚点：

```text
千兆铜口 PHY：
    常见于 ZYNQ PS GEM、MCU Ethernet、嵌入式 Linux 板卡。

SGMII PHY：
    常见于 Switch ASIC、SoC SerDes、多端口设计。

10G 光口 PHY/PCS：
    常见于 FPGA GT、SFP+、10GBASE-R。
```

迁移方法：

```text
1. 先看 MAC 支持哪些接口。
2. 再看板级连接和 IO 电压。
3. 再看 PHY strap 默认模式。
4. 再看驱动是否已有支持。
5. 最后补厂商扩展寄存器，例如 RGMII delay、LED、低功耗。
```

### Switch 芯片选择

先问：

```text
需要几个外部端口？
端口速率是百兆、千兆、2.5G、10G，还是混合？
CPU 用什么接口管理 Switch：MDIO、SPI、PCIe、内存映射总线？
是否需要 Linux DSA 支持，还是只能用厂商 SDK？
是否需要 VLAN、STP、LACP、ACL、QoS、IGMP Snooping、三层路由？
是否需要硬件时间戳、工业温度、环网协议？
```

Switch 芯片的关键不是“能不能转发”，而是：

```text
端口如何接 PHY/SerDes。
CPU Port 如何接主控。
控制报文如何 trap 到 CPU。
VLAN 和 MAC 表能否下发到硬件。
统计计数能否支撑现场排障。
```

### NIC 芯片选择

先问：

```text
主机侧是 PCIe、USB、SoC 内部总线还是 AXI？
目标吞吐是千兆、2.5G、10G、25G？
是否需要多队列、RSS、checksum offload、TSO、PTP？
操作系统是否已有稳定驱动？
是否需要 PXE、WoL、SR-IOV、DPDK？
```

学习 NIC 时要抓住：

```text
PCIe 枚举解决“主机是否看见设备”。
BAR/MMIO 解决“驱动如何配置设备”。
DMA ring 解决“包如何进出内存”。
中断/NAPI 解决“CPU 如何高效处理包”。
offload 解决“哪些工作交给硬件加速”。
```

## 硬件接口选择

| 接口 | 常见速率 | 用在哪里 | 关键风险 |
| --- | --- | --- | --- |
| MII | 10/100M | 老式 MAC-PHY | 管脚多，较少新设计使用 |
| RMII | 10/100M | 低成本 MCU/PHY | 50MHz 参考时钟来源 |
| RGMII | 10/100/1000M | ZYNQ/SoC 到千兆 PHY | DDR 时序、delay、电平 |
| SGMII | 10/100/1000M | Switch/SoC/PHY SerDes | SerDes lock、模式、自协商 |
| QSGMII | 4 路千兆 | Switch 多端口聚合接口 | lane 映射、SerDes 配置 |
| USXGMII | 多速率 | 2.5G/5G/10G PHY | PCS/速率协商复杂 |
| XGMII | 10G 逻辑接口 | 10G MAC 与 PCS | 常在 FPGA/IP 内部 |
| PCIe | 主机总线 | NIC、Switch 管理、高速 DMA | 枚举、BAR、MSI-X、DMA 一致性 |
| AXI-Stream | FPGA 数据流 | MAC、DMA、用户逻辑 | tlast/tkeep/tuser、背压 |

注意：这些是硬件接口，不是 TCP/IP 协议。

## 原理图检查总表

| 模块 | 检查项 |
| --- | --- |
| 电源 | 电压值、上电顺序、电流余量、模拟/数字电源隔离 |
| Reset | 极性、释放时序、是否由 GPIO 控制、是否满足 datasheet |
| 时钟 | 晶振频率、参考时钟方向、抖动要求、时钟使能 |
| Strap | PHY 地址、接口模式、delay、启动模式、LED 模式 |
| 管理总线 | MDIO/I2C/SPI 上拉、地址冲突、电平匹配 |
| MAC-PHY | RGMII/RMII/SGMII 模式、IO 电压、走线长度和约束 |
| 高速串行 | lane 极性、AC 耦合、电源噪声、参考时钟、SFP 控制脚 |
| RJ45 | 网络变压器、中心抽头、ESD、Bob Smith 端接、LED |
| PCIe | REFCLK、PERST#、lane 宽度、极性、电源管理 |

## 软件架构选择

### 裸机 lwIP

适合：

```text
资源有限。
启动快。
协议简单。
需要直接控制 DMA/cache。
```

典型项目：

```text
ZYNQ7010 TCP echo。
MCU UDP 控制。
小型工业设备。
```

### Linux 网络栈

适合：

```text
需要完整 TCP/IP 生态。
需要 SSH、HTTP、MQTT、SNMP、iptables、bridge。
需要成熟网卡驱动或 Switch DSA。
```

典型项目：

```text
嵌入式网关。
多端口交换设备。
PCIe NIC 网卡。
工业控制主机。
```

### FPGA Firmware + RTL

适合：

```text
10GbE 或更高速率。
固定 UDP/裸 Ethernet。
低延迟或线速处理。
PL 数据源直接上网。
```

典型项目：

```text
10G UDP 数据采集。
硬件包过滤。
时间同步采集系统。
```

## 项目迁移方法

从一个示例迁移到自己的板卡时，按这个顺序改：

```text
1. 换硬件连接：确认 PHY/Switch/NIC 接口和地址。
2. 换时钟/reset：确认启动时序和 GPIO。
3. 换驱动绑定：修改 device tree、BSP、base address 或 SDK 初始化。
4. 换链路配置：phy-mode、速率、duplex、RGMII delay。
5. 换网络参数：MAC、IP、VLAN、端口、应用协议。
6. 换验证脚本：ping、iperf、tcpdump、ethtool、寄存器读取。
```

不要直接复制示例工程后只改 IP 地址。以太网项目最常见的问题通常在硬件接口和底层初始化，而不是应用层。

## 本章总结

完整项目的学习重点不是“某个函数怎么写”，而是形成端到端判断力：

```text
硬件是否能工作。
芯片是否能被读到。
链路是否 up。
帧是否正确收发。
协议是否完成交互。
应用是否按预期处理。
出现问题时证据在哪一层。
```
