# 05 调试排障与学习路线

## 总原则

以太网排障要从底往上，不要一开始就看 TCP。

```text
电源/时钟/reset
  |
芯片管理接口
  |
PHY link
  |
MAC 帧收发
  |
二层交换/VLAN
  |
ARP/ND
  |
IP/ICMP
  |
UDP/TCP
  |
应用
```

每一层都要找证据。没有证据就不要跳层。

## 一页速查表

| 现象 | 优先检查 | 常见原因 |
| --- | --- | --- |
| PHY ID 读不到 | 电源、reset、MDC/MDIO、PHY 地址 | strap 错、reset 未释放、MDIO 上拉缺失 |
| link 不 up | 网线、对端、参考时钟、自协商 | 速率不匹配、SGMII/RGMII 模式错 |
| link up 但 ARP 不通 | MAC 收发、VLAN、PVID、STP | VLAN 不同、端口 blocked、MAC 未开 |
| ARP 通但 ping 不通 | IP、ICMP、FCS、ACL | IP 错、防火墙、包损坏 |
| ping 通但 TCP 不通 | 端口、listen、防火墙、回调 | 服务未启动、tcp_bind 失败 |
| TCP 通但吞吐低 | MTU、窗口、DMA、CPU、队列 | 小包 PPS 太高、offload 关闭 |
| 只有千兆不通 | RGMII delay、时钟、走线 | 100M 正常但 1000M 时序边界小 |
| VLAN 不通 | PVID、tag/untag、trunk allowed | 出口 tag 模式和对端不一致 |
| Switch CPU 高 | trap、广播风暴、未知单播 | 控制报文未限速、环路 |

## 工具链

硬件工具：

```text
万用表：确认电源和 reset 电平。
示波器：确认参考时钟、reset 时序、MDC。
逻辑分析仪：低速 MDIO/I2C/SPI 管理接口。
高速示波器/误码仪：SerDes、10G、眼图和误码。
```

软件工具：

```text
串口日志：裸机/ZYNQ/Firmware bring-up。
Wireshark/tcpdump：ARP、ICMP、TCP/UDP 抓包。
ethtool：Linux NIC/PHY 状态和统计。
bridge/fdb/vlan 命令：Linux switch/bridge 状态。
ILA：FPGA 内部 AXI-Stream、MAC、DMA 信号。
```

## 学习路线

### 第一阶段：能把链路点亮

掌握：

```text
PHY 供电、reset、时钟。
MDIO 读 PHY ID。
link up、speed、duplex。
RGMII/RMII/SGMII 区别。
```

练习：

```text
读一次 PHY ID。
读 link 状态。
强制 100M/1000M 对比现象。
观察 link LED 和寄存器是否一致。
```

### 第二阶段：能解释一次 ping

掌握：

```text
以太网帧。
ARP。
IPv4。
ICMP。
MAC 地址表。
```

练习：

```text
清空 PC ARP 表。
打开抓包。
ping 板卡。
解释 ARP request、ARP reply、ICMP request、ICMP reply 每一帧。
```

### 第三阶段：能配置交换芯片

掌握：

```text
Access、Trunk、PVID、Tagged/Untagged。
MAC learning。
STP forwarding/blocking。
CPU port。
端口统计和 MAC 表。
```

练习：

```text
配置两个端口同 VLAN 互通。
配置一个 trunk 口带 tag。
查看 MAC 表学习。
故意改错 PVID，观察 ARP 失败。
```

### 第四阶段：能理解 NIC 驱动

掌握：

```text
PCIe BAR。
DMA descriptor ring。
RX/TX buffer。
中断和 NAPI。
checksum/VLAN/RSS offload。
```

练习：

```text
用 ethtool 查看 offload。
调整 MTU。
观察 ethtool -S 统计。
用 iperf 测吞吐。
关闭 checksum offload 对比抓包。
```

### 第五阶段：能做 FPGA/ZYNQ 网络数据面

掌握：

```text
PS GEM + lwIP。
PL 数据源 + PS 协议栈。
PL AXI Ethernet。
10GbE MAC/PCS/PMA。
AXI-Stream、DMA、cache。
```

练习：

```text
跑通 ZYNQ TCP echo。
把 PL 采集数据通过 UDP 发出。
用 ILA 看 AXI-Stream tvalid/tready/tlast。
用 iperf 或自定义 UDP 测吞吐。
```

## 建议的知识闭环

每学一个概念，都按这五步闭环：

```text
1. 它解决什么问题？
2. 它在哪一层？
3. 它在硬件里对应哪个模块？
4. 它在软件里对应哪个寄存器、驱动函数或命令？
5. 它出错时抓包或计数器会看到什么？
```

例如 RGMII：

```text
解决 MAC 和 PHY 千兆并行传输。
属于 MAC-PHY 接口，不是 TCP/IP 协议。
硬件对应 TXD/RXD/TXC/RXC/TX_CTL/RX_CTL。
软件对应 Vivado GEM 配置、device tree phy-mode、PHY delay 寄存器。
出错常见 FCS error、100M 通 1000M 不通、ping 丢包。
```
