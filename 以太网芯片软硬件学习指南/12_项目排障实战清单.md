# 12 项目排障实战清单

## 使用方法

这份清单按故障现象组织。排障时不要跳层，按证据逐层推进：

```text
电源/时钟/reset
  |
芯片 ID
  |
link
  |
MAC 帧
  |
VLAN/二层
  |
ARP/IP/ICMP
  |
UDP/TCP
  |
应用协议
```

每一步至少拿到一个证据：示波器波形、寄存器、日志、计数器、抓包或命令输出。

## 1. PHY ID 读不到

现象：

```text
MDIO read 返回 0xffff、0x0000 或超时。
驱动报 cannot find PHY。
xemac_add() 失败。
```

排查顺序：

```text
1. PHY 电源是否正确。
2. reset 是否释放。
3. 参考时钟是否存在。
4. MDC 是否有波形。
5. MDIO 是否有上拉。
6. PHY 地址 strap 是否正确。
7. MAC/MDIO 控制器是否选对。
8. 多 PHY 是否地址冲突。
```

证据：

```text
示波器看 MDC。
逻辑分析仪看 MDIO transaction。
读取 PHY ID1/ID2。
串口打印扫描到的 PHY 地址。
```

## 2. link 不 up

现象：

```text
link LED 不亮。
ethtool 显示 Link detected: no。
PHY BMSR link bit 为 0。
Switch 端口 down。
```

排查：

```text
网线、光模块、对端端口。
PHY reset 和参考时钟。
自协商是否开启。
强制速率是否双方一致。
RGMII/SGMII/USXGMII 模式是否一致。
SFP+ LOS/TX_DISABLE/MOD_ABS。
SerDes PLL lock、block lock。
```

常见误区：

```text
MDIO 能读 PHY ID 不代表 link 一定能 up。
link up 不代表 RGMII 时序一定正确。
```

## 3. link up 但 ARP 不通

现象：

```text
PC 发 ARP request。
设备没有 ARP reply。
交换机 MAC 表没有学习到设备 MAC。
```

排查：

```text
MAC RX 计数是否增加。
FCS/CRC error 是否增加。
Switch VLAN/PVID 是否正确。
STP 状态是否 forwarding。
xemacif_input() 是否持续调用。
MAC 地址过滤是否丢弃广播。
```

抓包：

```text
PC 侧能看到 ARP request 发出。
设备侧或镜像口能否看到 ARP request 到达。
设备是否发出 ARP reply。
```

判断：

```text
设备收不到 ARP：
    查 PHY/MAC/Switch/VLAN。

设备收到但不回：
    查 IP、协议栈、netif、ARP 处理。

设备回了但 PC 收不到：
    查交换机转发、VLAN、MAC 表、PC 防火墙或网卡。
```

## 4. VLAN 不通

现象：

```text
同一物理交换机上部分端口能通，部分不能通。
抓包看到 VLAN tag 不符合预期。
MAC 表中 VID 错误。
```

排查：

```text
入口 PVID。
VLAN member。
出口 tagged/untagged。
Trunk allowed VLAN。
CPU Port tag 模式。
Linux bridge vlan_filtering。
Switch STP state。
```

命令：

```bash
bridge vlan show
bridge fdb show
tcpdump -i <iface> -nn -e vlan
```

原则：

```text
Access 口通常收 untagged，内部归入 PVID，出方向剥离 tag。
Trunk 口通常允许多个 VLAN，出方向保留 tag。
CPU Port 常需要 tagged，方便 Linux 区分 VLAN。
```

## 5. ping 通但 TCP 连不上

现象：

```text
ping 正常。
telnet/nc/Test-NetConnection 指定端口失败。
抓包只有 SYN，没有 SYN ACK。
```

排查：

```text
服务是否启动。
端口是否 bind 成功。
防火墙是否允许。
TCP listen backlog。
lwIP tcp_accept 是否注册。
应用主循环是否处理 input/timer。
```

抓包判断：

```text
SYN -> 无响应：
    服务未运行、防火墙丢弃、设备未收到。

SYN -> RST：
    主机收到但端口未监听或主动拒绝。

SYN -> SYN ACK -> ACK 后断开：
    应用层或协议处理异常。
```

## 6. TCP 慢或卡顿

现象：

```text
TCP 能连上，但吞吐低。
有大量 retransmission。
发送一段时间后卡住。
```

排查：

```text
TCP window。
MSS/MTU。
应用是否及时读取。
lwIP tcp_recved 是否调用。
tcp_sndbuf 是否不足。
DMA/cache 是否正确。
NIC offload 是否开启。
Switch 队列是否丢包。
```

工具：

```bash
iperf3 -c <ip> -P 4
tcpdump -i eth0 -nn tcp
ethtool -S eth0
```

lwIP 注意：

```text
收到 pbuf 后要 pbuf_free。
处理完接收数据要 tcp_recved。
tcp_write 返回 ERR_MEM 时要等 tcp_sent 回调继续发。
```

## 7. UDP 丢包

现象：

```text
UDP 高速发送时应用层序号跳变。
PC 网卡 RX dropped 增加。
FPGA FIFO overflow。
```

排查：

```text
发送速率是否超过接收能力。
MTU 是否匹配。
是否使用 jumbo frame。
PC 应用读取是否足够快。
NIC ring 是否太小。
Switch buffer 是否丢包。
FPGA tready 背压是否处理。
```

建议：

```text
UDP payload 增加 sequence。
统计发送包数、接收包数、丢包数。
先低速稳定，再逐步升速。
```

## 8. 10G block lock 失败

现象：

```text
PCS block lock 为 0。
hi BER 置位。
PC/Switch 10G link 不 up。
```

排查：

```text
GT refclk 频率和质量。
GT reset 顺序。
lane 极性是否反。
SFP+ 模块速率。
光纤或 DAC 是否匹配。
TX_DISABLE 是否释放。
PCS/PMA 配置是否为 10GBASE-R。
```

ILA 信号：

```text
gt_pll_lock
tx_reset_done
rx_reset_done
block_lock
hi_ber
rx_valid
```

## 9. DMA 卡死

现象：

```text
DMA busy 一直不清。
TX/RX descriptor 不回收。
网络收发停住。
```

排查：

```text
descriptor owner 位。
buffer 地址对齐。
DMA 地址是否可访问。
cache flush/invalidate。
AXI-Stream tlast 是否出现。
tkeep 是否正确。
中断是否触发。
BD ring head/tail 是否推进。
```

FPGA 常见原因：

```text
没有 tlast，DMA 不知道包结束。
tvalid 拉高但 tready 不处理。
包长度和 descriptor 长度不一致。
跨时钟 FIFO underflow/overflow。
```

## 10. Switch CPU 高

现象：

```text
管理页面卡顿。
SSH 慢。
CPU 中断高。
数据转发异常。
```

排查：

```text
广播风暴。
未知单播大量 flood。
控制报文 trap 未限速。
STP 环路。
ACL 把大量流量 redirect to CPU。
硬件转发未生效，走软件慢路径。
```

证据：

```text
CPU port RX 计数。
trap/drop 计数。
MAC 表是否抖动。
STP topology change。
端口广播/组播计数。
```

## 11. 现场交付排障顺序

现场人员可按这张短表执行：

```text
1. 看 Power/Run/Error LED。
2. 看 Link/Activity LED。
3. 确认 IP、Mask、Gateway。
4. ping 设备 IP。
5. arp -a 查看 MAC。
6. 测 TCP/UDP 端口。
7. 抓包看 ARP、ICMP、TCP/UDP。
8. 查看设备日志和端口统计。
9. 更换网线、端口、交换机。
10. 恢复出厂或导出配置。
```

## 总结

以太网项目排障的核心不是记住所有命令，而是坚持分层证据：

```text
没有芯片 ID，就不要看 link。
没有 link，就不要看 ARP。
没有 ARP，就不要看 TCP。
没有 TCP，再看应用协议。
```
