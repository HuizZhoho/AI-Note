# TCP/IP 抓包与排查速查

这份速查表用于实操排错。建议配合主文档 `README.md` 使用。

## 1. 常用命令

### 1.1 查看 IP 配置

Windows：

```powershell
ipconfig /all
```

重点看：

```text
IPv4 地址
IPv6 地址
子网掩码
默认网关
DNS 服务器
DHCP 是否启用
网卡名称
```

Linux：

```bash
ip addr
ip route
```

### 1.2 测试连通性

Windows：

```powershell
ping 192.168.1.1
ping example.com
ping -6 example.com
```

含义：

```text
ping 网关：
    测本地网络和网关是否通。

ping 目标 IP：
    测 IP 层是否可达。

ping 域名：
    同时涉及 DNS 和 IP 连通性。
```

### 1.3 查看路径

Windows：

```powershell
tracert example.com
```

Linux/macOS：

```bash
traceroute example.com
```

用途：

```text
查看数据包经过哪些路由节点。
定位在哪一跳开始不通或延迟变大。
```

### 1.4 查看 DNS

Windows：

```powershell
nslookup example.com
ipconfig /displaydns
ipconfig /flushdns
```

Linux：

```bash
nslookup example.com
dig example.com
```

### 1.5 查看 ARP 缓存

Windows：

```powershell
arp -a
```

Linux：

```bash
ip neigh
```

用途：

```text
确认 IPv4 地址是否已经解析到 MAC 地址。
```

### 1.6 查看 TCP/UDP 连接和监听端口

Windows：

```powershell
netstat -ano
netstat -ano | findstr ":80"
```

Linux：

```bash
ss -tunap
```

重点看：

```text
LISTEN：
    服务正在监听。

ESTABLISHED：
    TCP 连接已建立。

SYN_SENT：
    SYN 已发出，等待对端回应。

TIME_WAIT：
    连接关闭后的等待状态。
```

## 2. Wireshark 过滤表达式

### 2.1 链路层

```text
eth
eth.addr == 00:11:22:33:44:55
eth.src == 00:11:22:33:44:55
eth.dst == ff:ff:ff:ff:ff:ff
eth.type == 0x0800
eth.type == 0x86dd
```

### 2.2 ARP

```text
arp
arp.opcode == 1
arp.opcode == 2
arp.src.proto_ipv4 == 192.168.1.100
arp.dst.proto_ipv4 == 192.168.1.1
```

判断：

```text
只有 Request 没有 Reply：
    目标不在线、不同网段、网卡抓错、链路层异常。
```

### 2.3 IPv4 / IPv6

```text
ip
ip.addr == 192.168.1.100
ip.src == 192.168.1.100
ip.dst == 8.8.8.8

ipv6
ipv6.addr == fe80::1
```

### 2.4 ICMP / ICMPv6

```text
icmp
icmp.type == 8
icmp.type == 0

icmpv6
icmpv6.type == 135
icmpv6.type == 136
```

含义：

```text
ICMP type 8：
    Echo Request。

ICMP type 0：
    Echo Reply。

ICMPv6 type 135：
    Neighbor Solicitation。

ICMPv6 type 136：
    Neighbor Advertisement。
```

### 2.5 DNS

```text
dns
dns.qry.name == "example.com"
udp.port == 53
tcp.port == 53
```

判断：

```text
有 DNS Query 没有 Response：
    DNS 服务器不可达、被防火墙拦截、网络断开。

DNS 返回 NXDOMAIN：
    域名不存在或拼写错误。
```

### 2.6 TCP

```text
tcp
tcp.port == 80
tcp.port == 443
tcp.flags.syn == 1
tcp.flags.fin == 1
tcp.flags.reset == 1
tcp.analysis.retransmission
tcp.analysis.duplicate_ack
```

判断：

```text
只有 SYN，没有 SYN ACK：
    对端不可达、端口未开、防火墙丢包。

收到 RST：
    对端拒绝连接，常见于端口未监听或应用主动重置。

大量 retransmission：
    丢包、链路质量差、拥塞或 MTU 问题。
```

### 2.7 UDP

```text
udp
udp.port == 53
udp.port == 67
udp.port == 68
udp.port == 123
```

### 2.8 HTTP / TLS

```text
http
http.request
http.response
http.response.code == 404

tls
tcp.port == 443
```

注意：

```text
HTTPS 的 HTTP 内容被 TLS 加密，普通抓包通常看不到明文 URL 路径和响应正文。
```

## 3. 典型故障定位流程

### 3.1 完全不能上网

```text
检查网线/Wi-Fi 是否连接
  |
ipconfig /all 查看是否有 IP
  |
没有 IP：查 DHCP
  |
有 IP：ping 默认网关
  |
网关不通：查本地网络、ARP、交换机、路由器
  |
网关通：ping 公网 IP
  |
公网 IP 不通：查路由、运营商、出口防火墙
  |
公网 IP 通但域名不通：查 DNS
```

### 3.2 能 ping IP，不能 ping 域名

```text
ping 8.8.8.8 成功
  |
ping example.com 失败
  |
nslookup example.com
  |
无响应：DNS 服务器问题
  |
返回错误：域名或 DNS 解析问题
```

### 3.3 ping 通但 TCP 连接失败

```text
ping 目标 IP 成功
  |
抓 tcp.flags.syn == 1
  |
只有 SYN 无回应：防火墙/路由/服务端未通
  |
收到 RST：端口未监听或服务拒绝
  |
三次握手成功但应用无响应：应用层协议或服务异常
```

### 3.4 能打开部分网站，部分打不开

可能原因：

```text
DNS 解析差异
IPv6 优先但 IPv6 路径异常
MTU 问题
TLS 证书问题
代理问题
运营商或防火墙策略
```

排查：

```text
nslookup 目标域名
ping -4 目标域名
ping -6 目标域名
tracert 目标域名
抓 DNS、TCP、TLS
```

### 3.5 小包正常，大包异常

可能原因：

```text
MTU 不匹配
分片被阻止
隧道/VPN 额外封装导致实际 MTU 变小
路径 MTU 发现失败
```

排查方向：

```text
测试不同大小 ping
观察 ICMP fragmentation needed
检查 VPN/PPPoE/隧道配置
```

## 4. 分层排查口诀

```text
先看物理：
    网线、Wi-Fi、link 状态。

再看地址：
    IP、掩码、网关、DNS。

再看邻居：
    ARP 或 NDP 是否成功。

再看路由：
    默认路由和下一跳是否正确。

再看传输：
    TCP 是否握手，UDP 是否有响应。

最后看应用：
    DNS、HTTP、TLS、服务状态。
```
