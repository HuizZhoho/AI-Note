# 04 全双工、流控、链路聚合

## 本章解决什么问题

本章解释交换链路性能相关的三个主题：

```text
全双工：
    为什么现代交换网络可以同时收发。

流控：
    接收方或交换机来不及处理时怎么办。

链路聚合：
    如何把多条物理链路组成一个逻辑链路。
```

## 全双工

### 通俗理解

半双工像对讲机：

```text
同一时间只能一个人说。
如果两个人同时说，会冲突。
```

全双工像电话：

```text
双方可以同时说和听。
```

现代交换机端口通常是全双工。全双工环境下，每个端口是一条点到点链路，不再需要共享介质上的 CSMA/CD 冲突检测。

### 工程影响

全双工带来：

```text
收发方向独立。
理论上 1G 全双工可同时 1G 收 + 1G 发。
不再有传统共享以太网碰撞域。
链路距离不再受半双工冲突检测窗口限制。
```

常见故障是 duplex mismatch：

```text
一端全双工，一端半双工。
```

症状：

```text
ping 可能通。
吞吐很差。
有大量 CRC、late collision、丢包。
大流量时特别明显。
```

## 流控

### 为什么需要流控

交换机内部有缓冲区，但缓冲区有限。

当多个入口同时向一个出口发流量：

```text
入口总速率 > 出口速率
```

就会产生拥塞。处理方式有：

```text
排队
丢包
通知对端暂停
```

### PAUSE 帧

IEEE 802.3x PAUSE 是以太网流控机制。

基本思想：

```text
接收方快撑不住了
  |
发送 PAUSE 控制帧
  |
对端暂停一段时间
```

PAUSE 的问题：

```text
它通常暂停整个链路，而不是只暂停某一条业务流。
可能导致无关流量也被影响。
在数据中心中需要谨慎使用。
```

### 背压

半双工环境里，有时通过制造拥塞或冲突信号让对端降低发送，这类机制可理解为背压。现代全双工交换网络里更常见的是 PAUSE 或更高级的拥塞管理。

## 链路聚合

### 它解决什么问题

链路聚合把多条物理链路组成一个逻辑链路：

```text
Switch A ==多条物理链路== Switch B
        \__ 一个逻辑聚合组 __/
```

收益：

```text
增加总体带宽。
提供链路冗余。
避免 STP 把多条并行链路阻塞掉。
```

### LACP

LACP 是 Link Aggregation Control Protocol。

它用于：

```text
协商哪些端口能加入聚合组。
检测链路和对端信息是否一致。
避免错误聚合。
```

静态聚合不跑 LACP，配置简单但风险更高；动态聚合使用 LACP，更适合生产网络。

### 为什么单条流量不能简单翻倍

链路聚合通常按哈希分流。

哈希输入可能包括：

```text
源 MAC
目的 MAC
源 IP
目的 IP
源端口
目的端口
```

同一条 TCP 连接通常会固定走同一条物理链路，避免乱序。因此：

```text
多条流可以分摊到多条链路。
单条流通常不会被拆到多条链路上。
```

这就是为什么 4 条 1G 聚合后，总带宽可能接近 4G，但单个 TCP 下载不一定超过 1G。

## 常见误区

```text
误区 1：全双工等于双倍业务速度。
    它表示收发方向可同时工作，不代表单方向翻倍。

误区 2：PAUSE 一定能解决丢包。
    PAUSE 可能把拥塞传播到上游，甚至影响无关流量。

误区 3：LACP 会把一个包拆到多条链路。
    通常不会。它按流分担，不按包随意分担。

误区 4：链路聚合可以替代所有冗余设计。
    它适合点到点多链路，不等于三角拓扑无环方案。
```

## 排障视角

全双工问题：

```text
查看端口速率和双工。
查看 CRC、collision、late collision。
统一两端自协商或手动配置。
```

流控问题：

```text
查看 PAUSE 帧计数。
查看队列丢包。
确认是否存在多入口打一出口。
```

LACP 问题：

```text
两端聚合模式是否一致。
成员链路速率/双工是否一致。
VLAN trunk 配置是否一致。
LACP system ID、key、actor/partner 状态是否正常。
```
