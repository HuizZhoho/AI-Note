# 03 环路与生成树协议

## 本章解决什么问题

二层交换网络如果形成环路，会出现严重问题：

```text
广播风暴
未知单播风暴
MAC 地址表震荡
重复帧
CPU 和链路被打满
```

生成树协议 STP 的目标是：

```text
允许物理上有冗余链路。
逻辑上阻塞部分链路。
保证二层转发拓扑无环。
故障时重新启用备用链路。
```

## 初学者通俗理解

你希望城市道路有备用路，但如果所有路口都不管方向，车辆可能一直绕圈。STP 像交通管制：

```text
保留备用路。
平时关闭某些路口。
主路坏了，再打开备用路。
```

代价是部分链路平时不能转发普通数据，但换来稳定。

## 为什么二层环路危险

IP 包有 TTL，经过路由器会递减，最终会被丢弃。以太网帧没有类似 TTL 的字段。

如果广播帧进入二层环路：

```text
交换机 A 泛洪到 B
B 泛洪到 C
C 又泛洪回 A
A 再泛洪
```

结果：

```text
帧不会自然消失。
广播会被不断复制。
MAC 表会在多个端口间震荡。
网络很快不可用。
```

## STP 基本概念

STP 通过 BPDU 交换拓扑信息。

核心对象：

```text
Bridge ID：
    交换机身份，通常由优先级 + MAC 构成。

Root Bridge：
    生成树的根交换机。

Root Path Cost：
    到根桥的路径成本。

BPDU：
    交换机之间用于生成树计算的控制报文。
```

## STP 如何选择拓扑

简化流程：

```text
1. 选 Root Bridge。
2. 每台非根交换机选一个 Root Port。
3. 每个链路段选一个 Designated Port。
4. 其余可能造成环路的端口进入阻塞状态。
```

端口角色：

```text
Root Port：
    非根交换机到根桥的最佳路径端口。

Designated Port：
    某个网段上负责转发到根方向的端口。

Blocked/Alternate：
    为避免环路而不转发普通数据的端口。
```

## 端口状态

传统 STP 常见状态：

```text
Blocking：
    不转发数据，只听 BPDU。

Listening：
    参与拓扑计算，不学习 MAC。

Learning：
    学习 MAC，不转发普通数据。

Forwarding：
    学习 MAC，也转发普通数据。

Disabled：
    管理或物理关闭。
```

这些状态转换带来收敛时间。传统 STP 收敛较慢。

## RSTP

RSTP 是 Rapid Spanning Tree Protocol，目标是更快收敛。

重要变化：

```text
端口状态简化：
    Discarding、Learning、Forwarding。

端口角色更明确：
    Root、Designated、Alternate、Backup。

边缘端口：
    连接终端设备时可以快速进入转发。
```

RSTP 的工程意义：

```text
链路故障时恢复更快。
终端接入口不必等待传统 STP 的长时间状态转换。
```

## MSTP

MSTP 是 Multiple Spanning Tree Protocol。

它解决的问题：

```text
一个 STP 实例控制所有 VLAN：
    简单但链路利用率差。

每个 VLAN 一个 STP：
    灵活但实例太多。

MSTP：
    把多个 VLAN 映射到少量生成树实例。
```

MSTP 适合 VLAN 多、拓扑复杂的企业网络。

## STP 为什么牺牲链路

很多初学者会疑惑：

```text
买了两条链路，为什么 STP 要阻塞一条？
```

原因是二层没有 TTL，一旦环路转发，后果比浪费链路更严重。STP 的设计优先级是：

```text
先保证无环和稳定。
再考虑利用率。
```

如果想同时利用多条链路，应使用 LACP 等链路聚合，而不是让二层形成无控制环路。

## BPDU 和保护机制

工程中常见保护：

```text
PortFast / Edge Port：
    接终端的端口快速转发。

BPDU Guard：
    边缘端口收到 BPDU 就关闭，防止误接交换机。

Root Guard：
    防止非预期交换机成为 Root Bridge。

Loop Guard：
    防止单向链路导致阻塞端口错误转发。
```

## 排障视角

典型症状：

```text
全网突然变慢：
    可能是广播风暴或环路。

MAC 地址在端口间跳变：
    可能存在二层环路。

某条链路不转发：
    可能被 STP 阻塞，不一定是故障。

插入小交换机后网络异常：
    可能引入环路或非法 BPDU。
```

排查顺序：

```text
1. 查看 STP root bridge 是否符合规划。
2. 查看端口角色和状态。
3. 查看拓扑变化计数。
4. 查看是否有 BPDU Guard、Root Guard 触发。
5. 查看广播/未知单播流量是否异常。
```
