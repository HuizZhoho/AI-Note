# 01 网络与二层交换基础

## 本章解决什么问题

本章回答三个基础问题：

```text
网络为什么要分层？
二层交换到底工作在哪一层？
以太网帧、MAC 地址、MTU 和交换机之间是什么关系？
```

二层交换机的世界不是 IP 地址优先，而是 MAC 地址和以太网帧优先。理解这一点，后面的透明桥、VLAN、STP、LACP、QoS 都会清晰很多。

## 初学者通俗理解

可以把一台交换机想成楼里的快递分拣柜：

```text
端口：
    每个房间门口。

MAC 地址：
    每个住户的门牌。

以太网帧：
    快递包裹。

MAC 地址表：
    快递员记下“某个住户在哪个门口”。

泛洪：
    不知道住户在哪，就每个门口都问一遍。
```

交换机不是简单把所有信号复制到所有端口的集线器。交换机会学习、查表、过滤、转发，所以它能把不同端口上的通信隔离开，提高总体吞吐。

## 网络分层

常见 OSI 七层：

```text
7 应用层：HTTP、DNS、SNMP
6 表示层：编码、压缩、加密
5 会话层：会话维护
4 传输层：TCP、UDP
3 网络层：IPv4、IPv6、路由
2 数据链路层：Ethernet、MAC、VLAN、STP
1 物理层：电信号、光信号、网线、PHY
```

交换机主要工作在二层，也就是数据链路层。三层交换机还会做 IP 路由，但二层交换的基础仍然是 Ethernet frame 和 MAC 地址表。

## Ethernet 帧

简化的以太网帧：

```text
目的 MAC
源 MAC
EtherType / Length
Payload
FCS
```

常见 EtherType：

```text
0x0800  IPv4
0x0806  ARP
0x86DD  IPv6
0x8100  802.1Q VLAN Tag
```

交换机一般先看目的 MAC，再结合 VLAN 信息决定怎么转发。普通二层交换机不会关心 TCP 端口，也不会理解 HTTP 请求内容。

## MAC 地址

MAC 地址常见写法：

```text
00:11:22:33:44:55
```

类型：

```text
单播：
    发给某一台设备。

广播：
    ff:ff:ff:ff:ff:ff，发给同一广播域内所有设备。

组播：
    发给一组设备。
```

关键点：

```text
源 MAC 用来学习地址表。
目的 MAC 用来决定转发方向。
广播和未知目的通常会被泛洪。
```

## MTU

MTU 是链路层一次能承载的最大上层数据大小。普通以太网常见 MTU 是：

```text
1500 bytes
```

加入 VLAN tag 后，帧会多 4 字节。工程上要注意：

```text
设备是否支持带 VLAN 的帧长度。
隧道、VPN、PPPoE 会降低有效 MTU。
小包正常、大包异常时，要怀疑 MTU。
```

## 二层、三层、四层的边界

```text
二层：
    MAC、Ethernet、VLAN、STP、交换机。

三层：
    IP、路由、网关、TTL/Hop Limit。

四层：
    TCP、UDP、端口。
```

二层交换机转发依据是：

```text
VLAN + 目的 MAC
```

路由器转发依据是：

```text
目的 IP + 路由表
```

## 常见误区

```text
误区 1：交换机根据 IP 转发。
    普通二层交换机主要根据 MAC 转发。

误区 2：MAC 地址只在设备出厂时有用。
    MAC 是二层转发的核心。

误区 3：VLAN 就是 IP 网段。
    VLAN 是二层广播域，IP 网段是三层地址规划。它们常常一一对应，但不是同一概念。

误区 4：ping 不通一定是交换机坏了。
    ping 涉及二层、三层、ICMP、防火墙、路由等多处。
```

## 抓包和排障视角

Wireshark 过滤：

```text
eth
eth.addr == 00:11:22:33:44:55
eth.type == 0x0800
eth.type == 0x8100
arp
```

排查顺序：

```text
1. 端口 link 是否 up。
2. 网卡 MAC 是否能在交换机 MAC 表中看到。
3. VLAN 是否正确。
4. ARP 是否有请求和回应。
5. 是否存在广播风暴或环路。
```
