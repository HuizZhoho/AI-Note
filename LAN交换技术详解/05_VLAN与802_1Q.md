# 05 VLAN 与 802.1Q

## 本章解决什么问题

VLAN 用来把一个物理交换网络划分成多个逻辑二层网络。

它解决：

```text
广播域隔离
部门/业务逻辑隔离
地址规划清晰化
跨交换机传递多个二层网络
减少不必要广播影响
```

## 初学者通俗理解

一台交换机像一栋楼。没有 VLAN 时，所有人都在同一个大房间里喊话。VLAN 像把楼分成多个会议室：

```text
VLAN 10：
    研发。

VLAN 20：
    财务。

VLAN 30：
    访客。
```

同一个 VLAN 内可以二层通信，不同 VLAN 之间默认不能二层互通，需要三层路由。

## VLAN 不是 IP 网段

VLAN 是二层概念，IP 网段是三层概念。

常见设计是一一对应：

```text
VLAN 10 -> 192.168.10.0/24
VLAN 20 -> 192.168.20.0/24
```

但本质上：

```text
VLAN 决定广播域。
IP 网段决定三层寻址。
```

## 802.1Q Tag

802.1Q 在以太网帧中插入 4 字节 VLAN Tag。

简化结构：

```text
目的 MAC
源 MAC
TPID 0x8100
TCI
原 EtherType
Payload
FCS
```

TCI 中包含：

```text
PCP：
    优先级，3 bit。

DEI：
    丢弃指示。

VID：
    VLAN ID，12 bit。
```

VLAN ID 常见范围：

```text
1 - 4094
```

## Access、Trunk、Tagged、Untagged、PVID

### Access 端口

通常接终端。

```text
进端口：
    未带 tag 的帧被归入该端口 PVID。

出端口：
    发给终端时通常去掉 tag。
```

### Trunk 端口

通常接交换机、路由器、防火墙、服务器虚拟化主机。

```text
一条链路承载多个 VLAN。
多数 VLAN 帧带 tag。
```

### PVID

PVID 是端口默认 VLAN。

```text
未带 tag 的入方向帧
  |
按 PVID 归入某个 VLAN
```

### Tagged 和 Untagged

```text
Tagged：
    帧携带 VLAN Tag。

Untagged：
    帧不携带 VLAN Tag。
```

一个端口对某个 VLAN 是 tagged 还是 untagged，决定该 VLAN 的帧出端口时是否保留 tag。

## VLAN 转发过程

交换机转发时，MAC 表上下文包含 VLAN：

```text
(VLAN ID, MAC Address) -> Port
```

所以同一个 MAC 在不同 VLAN 中是不同表项。

转发逻辑：

```text
入端口确定 VLAN
  |
学习源 MAC 到该 VLAN
  |
查找目的 MAC + VLAN
  |
只在同 VLAN 内转发或泛洪
  |
出端口根据 tagged/untagged 规则处理 tag
```

## 单臂路由

不同 VLAN 之间要互通，需要三层设备。

单臂路由：

```text
交换机 trunk 连接路由器
  |
路由器子接口分别对应 VLAN 10、20、30
  |
路由器在 VLAN 间转发 IP 包
```

现代网络中，常用三层交换机 SVI 实现 VLAN 间路由：

```text
interface vlan 10
interface vlan 20
```

## MST 与 VLAN

STP 如果只有一个实例，所有 VLAN 使用同一棵树，链路利用率可能差。

MSTP 可以把不同 VLAN 映射到不同生成树实例：

```text
VLAN 10,20 -> MST Instance 1
VLAN 30,40 -> MST Instance 2
```

这样既控制实例数量，又可以让不同 VLAN 使用不同链路。

## VLAN 为什么不是安全边界本身

VLAN 能隔离广播域，但安全还要靠：

```text
ACL
防火墙
端口安全
认证
DHCP Snooping
Dynamic ARP Inspection
正确的 trunk 配置
```

如果 trunk 配错、native VLAN 配错或设备存在漏洞，可能出现 VLAN hopping 等问题。

## 常见误区

```text
误区 1：同一交换机上的所有端口都能互通。
    不同 VLAN 默认二层不互通。

误区 2：Trunk 就是更快的链路。
    Trunk 在 VLAN 语境下表示承载多个 VLAN，不等于链路聚合。

误区 3：PVID 是出方向 tag。
    PVID 主要用于未打 tag 入方向帧的 VLAN 归属。

误区 4：native VLAN 不重要。
    native VLAN 配置不一致会导致流量错入 VLAN 或安全风险。
```

## 排障视角

检查项：

```text
端口 VLAN 模式：access 还是 trunk。
access VLAN 是否正确。
trunk allowed VLAN 是否包含目标 VLAN。
PVID/native VLAN 是否一致。
MAC 是否在正确 VLAN 学到。
三层网关/SVI 是否 up。
```

典型问题：

```text
同 VLAN 不通：
    查 access VLAN、端口状态、MAC 表。

跨 VLAN 不通：
    查网关、路由、ACL、防火墙。

某 VLAN 过 trunk 不通：
    查 allowed VLAN 和 tag 规则。
```
