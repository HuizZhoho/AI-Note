# 08 交换机排障方法

## 本章解决什么问题

交换网络故障经常表现为：

```text
一会儿通一会儿不通。
同 VLAN 不通。
跨 VLAN 不通。
某些业务慢。
广播异常多。
环路导致全网卡死。
某个端口丢包但链路看起来是 up。
```

本章给出一套从物理层到应用层的排障路径，避免凭感觉乱改配置。

## 系统排障思路

核心原则：

```text
先界定范围。
再分层定位。
每次只改一个变量。
改完验证。
保留现场证据。
```

不要一开始就重启设备或大范围改配置。很多二层问题需要看 MAC 表、STP 状态、端口计数、VLAN 表和抓包现场。

## 排障流程

```text
定义问题
  |
确认影响范围
  |
查看物理状态
  |
查看 VLAN 和端口配置
  |
查看 MAC 学习
  |
查看 STP 状态
  |
查看错误计数和流量
  |
抓包验证
  |
定位根因
  |
执行修复
  |
持续观察
```

## 物理层检查

检查项：

```text
端口是否 up。
速率是否正确。
双工是否匹配。
光模块是否兼容。
光功率是否正常。
网线质量是否合格。
错误包是否增长。
```

常见错误计数：

```text
CRC error：
    线缆、光模块、干扰、双工不匹配。

Late collision：
    半双工/双工不匹配或异常共享介质。

Input drops：
    入方向缓冲不足或芯片资源问题。

Output drops：
    出方向拥塞。
```

## VLAN 问题

常见症状：

```text
同一交换机上两台主机不通。
接到另一个交换机后不通。
某个 VLAN 跨 trunk 不通。
```

检查：

```text
接入口 VLAN 是否正确。
trunk 是否允许该 VLAN。
native VLAN/PVID 是否一致。
MAC 是否在正确 VLAN 学到。
三层网关是否存在。
ACL 是否阻断。
```

判断：

```text
同 VLAN 内不通：
    优先查二层 VLAN、端口、MAC 表。

跨 VLAN 不通：
    查三层网关、路由、ACL、防火墙。
```

## STP 问题

常见症状：

```text
链路 up 但不转发。
网络周期性中断。
MAC 地址频繁漂移。
广播流量异常。
插入小交换机后网络瘫痪。
```

检查：

```text
Root Bridge 是否正确。
端口角色是否符合预期。
端口状态是否 forwarding。
是否有 topology change。
是否触发 BPDU Guard/Root Guard。
是否存在单向链路。
```

处理建议：

```text
规划根桥和备根桥。
接终端端口启用 edge/portfast。
边缘端口启用 BPDU Guard。
不要随便把 STP 全局关闭。
```

## Duplex mismatch

这是经典故障。

表现：

```text
小流量似乎正常。
大流量很慢。
CRC、collision、late collision 增长。
应用超时。
```

排查：

```text
查看两端速率/双工。
优先让两端都自协商。
如果手动配置，必须两端一致。
检查网线和光模块。
```

## 广播风暴和未知单播

广播风暴可能来自：

```text
二层环路。
异常设备。
恶意攻击。
协议配置错误。
```

未知单播过多可能来自：

```text
MAC 表溢出。
单向流量。
老化时间过短。
攻击。
非对称路径。
```

排查：

```text
查看端口流量排名。
查看广播/组播/未知单播计数。
查看 MAC flapping。
抓包看源头。
临时隔离可疑端口。
```

## 抓包方法

交换环境中抓包常用：

```text
端口镜像。
TAP。
主机本机抓包。
交换机 CPU 抓包。
```

注意：

```text
镜像口可能丢包。
镜像方向要选对。
VLAN tag 可能被交换机处理后才镜像。
高速链路上普通电脑可能抓不全。
```

Wireshark 过滤：

```text
eth
vlan
stp
arp
icmp
tcp.analysis.retransmission
eth.dst == ff:ff:ff:ff:ff:ff
```

## 常用故障矩阵

| 现象 | 优先怀疑 | 关键检查 |
| --- | --- | --- |
| 端口不亮 | 物理层 | 线缆、光模块、速率、管理 shutdown |
| 同 VLAN 不通 | 二层配置 | access VLAN、MAC 表、STP 状态 |
| 跨 VLAN 不通 | 三层配置 | 网关、SVI、路由、ACL |
| 全网变慢 | 环路/风暴 | STP、广播计数、MAC flapping |
| 某端口慢 | 双工/错误包 | CRC、duplex、丢包、队列 |
| TCP 慢但 ping 正常 | 丢包/MTU/QoS | 重传、MSS、队列丢包 |
| 语音卡顿 | QoS/拥塞 | PCP/DSCP、队列、调度 |

## 排障习惯

```text
记录故障开始时间。
记录影响范围。
保存变更历史。
先看只读状态，再改配置。
改动前备份配置。
改动后持续观察计数器。
```

真正高效的网络排障，不是记住所有命令，而是按层次收敛问题范围。
