# 07 二层安全与管理

## 本章解决什么问题

交换机不只是转发设备，也是安全和运维的关键位置。本章覆盖：

```text
二层常见攻击
交换机安全控制
SNMP 管理
端口镜像
RMON/监控
```

## 二层安全为什么重要

很多人把安全只理解成防火墙和三层 ACL，但二层攻击可以直接破坏局域网基础：

```text
让交换机泛洪所有流量。
伪装网关实施中间人攻击。
让 STP 拓扑重新收敛。
跳入不该进入的 VLAN。
伪造 DHCP 服务器分配错误网关。
```

## 常见攻击

### MAC Flooding

攻击者发送大量伪造源 MAC 帧，填满交换机 MAC 表。

后果：

```text
交换机查不到正常目的 MAC。
未知单播被大量泛洪。
攻击者可能嗅探到更多流量。
```

防护：

```text
端口安全限制 MAC 数量。
静态 MAC 或 sticky MAC。
异常 MAC 学习速率告警。
```

### ARP Spoofing

攻击者伪造 ARP：

```text
告诉主机：我是网关。
告诉网关：我是主机。
```

后果：

```text
中间人攻击。
流量被窃听或篡改。
网络中断。
```

防护：

```text
DHCP Snooping 建立可信绑定表。
Dynamic ARP Inspection 校验 ARP。
静态 ARP 用于关键设备。
```

### DHCP Attack

常见两类：

```text
DHCP Starvation：
    消耗 DHCP 地址池。

Rogue DHCP：
    伪造 DHCP 服务器，分配错误网关/DNS。
```

防护：

```text
DHCP Snooping。
只允许可信端口发 DHCP Offer。
限制接入口 DHCP 请求速率。
```

### STP Attack

攻击者发送更优 BPDU，试图成为 root bridge。

后果：

```text
流量路径改变。
拓扑反复收敛。
可能造成中断。
```

防护：

```text
BPDU Guard。
Root Guard。
规划 Root Bridge 优先级。
边缘端口启用保护。
```

### VLAN Hopping

攻击者试图进入不属于自己的 VLAN。

常见原因：

```text
Trunk 协商配置不当。
Native VLAN 设计不当。
双重 VLAN tag 场景。
```

防护：

```text
用户端口强制 access。
关闭动态 trunk 协商。
native VLAN 不承载用户业务。
限制 trunk allowed VLAN。
```

## 交换机管理

### SNMP

SNMP 用于网络设备监控和管理。

核心对象：

```text
Manager：
    管理平台。

Agent：
    交换机上的代理。

MIB：
    管理对象数据库。

OID：
    某个管理对象的编号。
```

SNMP 可用于：

```text
读取端口状态。
读取流量计数。
读取错误包。
接收 Trap 告警。
```

建议：

```text
优先使用 SNMPv3。
限制管理源 IP。
不要使用默认 community。
```

### 端口镜像

端口镜像用于抓包：

```text
源端口/源 VLAN
  |
复制流量
  |
镜像目的端口
  |
接分析仪或抓包电脑
```

注意：

```text
镜像端口可能带宽不足。
镜像可能丢包。
镜像通常不适合长期全量监控大流量。
```

### RMON 和监控

RMON 提供更细的统计和历史信息，例如：

```text
端口统计
历史采样
告警阈值
主机流量
会话矩阵
包捕获
```

现代网络也常使用：

```text
Telemetry
NetFlow/sFlow
Syslog
集中日志平台
NMS/可观测性平台
```

## 常见误区

```text
误区 1：二层网络不需要安全。
    许多攻击发生在同一局域网内。

误区 2：VLAN 本身就是强安全边界。
    VLAN 是隔离机制，但仍需 ACL、防火墙和安全特性配合。

误区 3：端口镜像看到的就是绝对完整真实流量。
    高负载时镜像可能丢包，方向和过滤也可能配置不完整。

误区 4：SNMP 只是查看流量。
    SNMP 还可能修改配置，必须控制权限。
```

## 排障视角

安全排查：

```text
MAC 表是否异常增长。
是否有 MAC flapping。
ARP 表是否异常。
是否出现非法 DHCP Offer。
STP root 是否被改变。
trunk allowed VLAN 是否过宽。
```

管理排查：

```text
SNMP 是否能访问。
community/用户/权限是否正确。
ACL 是否允许管理平台访问。
Trap 是否送达。
镜像方向是否正确。
```
