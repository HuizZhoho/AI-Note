# LAN 交换技术详解

这套笔记是基于 `TheAllNewSwitchBook.pdf` 的目录主题整理出的中文原创学习讲义，目标是让初学者建立完整的 LAN Switching 知识体系。它不是原书翻译，也不是逐页复述，而是把二层交换、VLAN、生成树、链路聚合、QoS、安全、管理、排障和交换芯片内部实现重新组织成工程化学习材料。

## 学习目标

学完后你应该能解释：

```text
交换机为什么不是简单的多口网线。
交换机如何学习 MAC 地址。
未知单播、广播、组播为什么会被泛洪。
二层环路为什么会导致广播风暴。
STP/RSTP/MSTP 如何阻断环路。
Access、Trunk、PVID、Tagged、Untagged 的区别。
LACP 为什么不能让单条 TCP 流量简单翻倍。
QoS 为什么是排队和调度策略，不是凭空增加带宽。
MAC flooding、ARP spoofing、VLAN hopping、STP 攻击如何发生。
交换芯片内部如何完成接收、分类、查表、排队和转发。
```

## 全局知识地图

```text
终端主机
  |
Access 端口
  |
交换机接收路径
  |
解析 Ethernet/VLAN/优先级
  |
MAC 地址表查找
  |
决定转发、过滤、泛洪或本地处理
  |
交换矩阵/共享内存/队列
  |
调度和输出
  |
Trunk/Access 端口发出
```

交换机的核心工作可以浓缩为四句话：

```text
从源 MAC 学习入口端口。
按目的 MAC 决定出口端口。
不知道目的地就泛洪。
为了避免环路，必须引入 STP/RSTP/MSTP 等控制机制。
```
