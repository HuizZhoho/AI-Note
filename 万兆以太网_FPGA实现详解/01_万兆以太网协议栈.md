# 01 万兆以太网协议栈

## 本章解决什么问题

本章解释 10GbE 协议栈从光口到以太网帧的层次：

```text
PMD：
    光模块和物理介质。

PMA：
    高速串行收发、CDR、串并转换。

PCS：
    64b/66b 编码、block lock、对齐、控制块。

XGMII：
    MAC 和 PHY/PCS 之间的逻辑接口。

MAC：
    以太网帧、FCS、最小帧、IFG、PAUSE、统计。
```

## 10GbE 和千兆以太网的关键差异

千兆 RGMII 常见是并行接口加外部 PHY。10GbE 通常是高速串行链路：

```text
1G RGMII：
    125 MHz DDR 并行接口，外部 PHY 接 RJ45。

10GBASE-R：
    10.3125 Gbps 串行线路速率，64b/66b 编码，常配 SFP+。
```

为什么是 10.3125 Gbps，不是刚好 10 Gbps？

```text
64b/66b 编码有 66/64 的开销。
10 Gbps * 66 / 64 = 10.3125 Gbps。
```

## 10GBASE-R

10GBASE-R 是常见 10GbE PCS/PMA 体系，适用于光口和背板等场景。

它的核心：

```text
64b/66b 编码
scrambler 扰码
block lock
高频串行传输
控制块和数据块区分
```

通俗理解：

```text
MAC 看到的是连续的以太网帧。
PCS/PMA 负责把这些帧变成适合高速串行链路传输的比特流。
```

## 64b/66b 编码

64b/66b 把 64 bit 数据加上 2 bit sync header。

```text
01：
    数据块。

10：
    控制块。
```

作用：

```text
让接收端能找到块边界。
区分数据和控制信息。
降低编码开销。
配合扰码改善频谱特性。
```

如果接收端无法稳定识别 66 bit block，就会出现：

```text
block lock 失败
link 不 up
误码率高
```

## PCS / PMA / PMD

### PCS

PCS 是 Physical Coding Sublayer。

负责：

```text
64b/66b encode/decode
block lock
alignment
scrambler/descrambler
fault indication
```

### PMA

PMA 是 Physical Medium Attachment。

负责：

```text
串并转换
CDR 时钟数据恢复
高速收发器适配
电气串行接口
```

在 Xilinx FPGA 中，PMA 很大程度上落在 GT Transceiver 和相关 wrapper 中。

### PMD

PMD 是 Physical Medium Dependent。

负责具体介质相关部分：

```text
SFP+ 光模块
光纤
DAC 铜缆
连接器
光电转换
```

## XGMII

XGMII 是 10G MAC 和 PHY/PCS 之间的逻辑接口。

典型概念：

```text
64 bit data
8 bit control
156.25 MHz 时钟
```

吞吐计算：

```text
64 bit * 156.25 MHz = 10 Gbps
```

在 FPGA IP 中，你不一定直接看到 XGMII，很多 IP 会把 MAC 用户侧包装成 AXI-Stream，但内部仍然有类似 XGMII 的 MAC/PCS 边界。

## MAC 层

10G MAC 处理以太网帧层面的事情：

```text
前导码和 SFD
目的 MAC / 源 MAC
EtherType / Length
Payload
FCS/CRC
最小帧长度
最大帧长度
IFG
PAUSE/PFC
统计计数
```

通常 MAC 硬件会自动：

```text
TX 添加或检查 FCS
RX 检查 FCS
处理 preamble/SFD
统计错误帧
处理最小帧 padding
```

Firmware 一般不需要自己生成 FCS。自定义纯 RTL MAC 除外。

## Ethernet Frame

标准以太网帧逻辑结构：

```text
Destination MAC
Source MAC
EtherType / Length
Payload
FCS
```

常见 EtherType：

```text
0x0800  IPv4
0x86DD  IPv6
0x0806  ARP
0x8100  VLAN
```

## VLAN 和 Jumbo Frame

10GbE 常用于高吞吐场景，经常启用 Jumbo Frame。

```text
普通 MTU：
    1500 bytes。

Jumbo MTU：
    常见 9000 bytes 左右，具体看设备支持。
```

Jumbo Frame 的好处：

```text
降低每字节协议开销。
降低每秒包数压力。
提高大块数据传输效率。
```

代价：

```text
链路路径上所有设备都要支持。
buffer 要更大。
错误包重传成本更高。
```

## PAUSE 和 PFC

### 802.3x PAUSE

PAUSE 让对端暂停发送一段时间。

问题：

```text
它可能暂停整条链路，影响所有流量。
```

### PFC

PFC 是 Priority Flow Control，可按优先级暂停。

常见于数据中心无损以太网场景，例如 RoCE。

注意：

```text
PFC 不是万能。
配置不当会导致拥塞传播、死锁或抖动。
```

## 初学者最小掌握

```text
10GbE 光口数据不是直接进 MAC，而是经过 PMD/PMA/PCS。
10GBASE-R 使用 64b/66b，线路速率是 10.3125 Gbps。
XGMII 逻辑吞吐是 64bit * 156.25MHz。
MAC 负责以太网帧，PCS/PMA/PMD 负责高速物理链路。
Firmware 主要配置寄存器、检测状态、管理 DMA，不参与每个 bit 的高速处理。
```
