# 03 收发数据完整流程

## 本章解决什么问题

本章把 10GbE 的 RX 和 TX 过程完整串起来。目标是看到一个包时，你能知道它经过了哪些模块、每一步谁负责、出错时该查哪里。

## RX：从光口到应用缓冲区

完整路径：

```text
SFP+ 光模块
  |
GT Transceiver
  |
PCS/PMA
  |
10G MAC
  |
AXI-Stream RX
  |
FIFO / DMA S2MM
  |
DDR RX buffer
  |
Firmware interrupt/polling
  |
cache invalidate
  |
协议解析或应用处理
```

### 1. SFP+ 接收光信号

SFP+ 把光信号转换为高速差分电信号。

需要检查：

```text
RX_LOS 是否为无光
模块是否插入
光功率是否正常
速率是否匹配
```

### 2. GT 恢复时钟和数据

GT 从串行比特流中恢复时钟并做串并转换。

状态：

```text
QPLL/CPLL lock
CDR lock
RX reset done
```

如果 GT 不稳定，后面的 PCS 不可能稳定 block lock。

### 3. PCS/PMA 解码

PCS 查找 66bit block 边界，进行 64b/66b 解码和 descramble。

关键状态：

```text
block lock
high BER
remote/local fault
```

block lock 失败通常说明：

```text
信号质量差
参考时钟错误
速率不匹配
极性/复位配置错误
对端没有发正确 10GBASE-R 信号
```

### 4. MAC 检查以太网帧

MAC 处理：

```text
preamble/SFD
MAC frame boundary
FCS check
minimum/maximum length check
pause frame
statistics counter
```

错误帧可能被 MAC 丢弃，并增加统计计数。

### 5. AXI-Stream 输出

MAC 用户侧输出 AXI-Stream：

```text
tdata  = frame data
tkeep  = valid bytes
tvalid = data valid
tready = downstream ready
tlast  = end of frame
tuser  = error or metadata
```

如果 DMA/FIFO 拉低 `tready`，MAC 或 FIFO 会产生 backpressure。持续 backpressure 可能导致丢包。

### 6. DMA 写 DDR

DMA S2MM 把 AXI-Stream 写入 DDR buffer。

Firmware 需要提前准备 RX descriptor：

```text
buffer address
buffer length
ownership
status
```

### 7. Firmware 处理中断

DMA 收到完整帧后产生中断或状态更新。

Firmware：

```text
读取 descriptor status
确认实际包长
invalidate cache
把 buffer 交给协议栈或应用
重新投递 RX buffer
```

### RX 伪代码

```c
void rx_submit_buffers(void)
{
    for (int i = 0; i < RX_DESC_NUM; i++) {
        rx_desc[i].addr = rx_buffer[i];
        rx_desc[i].len = RX_BUF_SIZE;
        rx_desc[i].owner = DMA_OWNED;
    }

    dma_start_rx(rx_desc);
}

void dma_rx_isr(void)
{
    while (dma_rx_desc_done()) {
        desc = dma_get_completed_rx_desc();

        cache_invalidate(desc->addr, desc->actual_len);

        if (desc->status & RX_ERR) {
            stats.rx_error++;
        } else {
            process_rx_packet(desc->addr, desc->actual_len);
        }

        desc->owner = DMA_OWNED;
        dma_refill_rx_desc(desc);
    }
}
```

## TX：从应用缓冲区到光口

完整路径：

```text
应用数据 / 协议栈
  |
构造 Ethernet/IP/UDP/TCP frame
  |
DDR TX buffer
  |
cache flush
  |
DMA MM2S
  |
AXI-Stream TX
  |
10G MAC
  |
PCS/PMA
  |
GT Transceiver
  |
SFP+ 光模块
```

### 1. 应用准备数据

应用可能发送：

```text
裸 Ethernet frame
UDP packet
TCP stream
自定义二层协议
```

如果 Firmware 自己构造 UDP，需要填：

```text
Ethernet header
IPv4 header
UDP header
Payload
```

FCS 通常由 MAC 自动添加。

### 2. 写入 TX buffer

Firmware 把帧放入 DDR buffer。

注意：

```text
buffer 对齐
tkeep 对应最后一拍有效字节
帧长不能超过 MAC 配置最大值
Jumbo frame 要两端都支持
```

### 3. cache flush

CPU 写 buffer 后，DMA 读 DDR 前必须 flush：

```c
cache_flush(tx_buffer, frame_len);
```

否则 DMA 可能读到旧数据。

### 4. DMA 发送

TX descriptor 包含：

```text
buffer address
frame length
start/end of packet
control flags
status
```

### 5. MAC 发送

MAC 处理：

```text
插入 preamble/SFD
保证 IFG
添加 FCS
处理 PAUSE/PFC
更新统计
```

### 6. PCS/PMA/GT/SFP+

PCS 进行 64b/66b 编码，GT 串行发送到 SFP+。

### TX 伪代码

```c
int send_udp_frame(const u8 *payload, u16_t payload_len)
{
    u8 *buf = alloc_tx_buffer();
    u16_t frame_len;

    frame_len = build_ether_ipv4_udp(buf,
                                     local_mac,
                                     remote_mac,
                                     local_ip,
                                     remote_ip,
                                     local_port,
                                     remote_port,
                                     payload,
                                     payload_len);

    cache_flush(buf, frame_len);

    tx_desc.addr = buf;
    tx_desc.len = frame_len;
    tx_desc.flags = TX_SOP | TX_EOP;
    tx_desc.owner = DMA_OWNED;

    dma_submit_tx(&tx_desc);

    return 0;
}
```

## 包边界

以太网是按帧传输，AXI-Stream 上用 `tlast` 标记帧结束。

```text
一个 Ethernet frame
  |
多个 AXI-Stream beat
  |
最后一个 beat tlast = 1
```

如果 `tlast` 丢失或提前：

```text
DMA 可能卡死
包长错误
MAC 发送异常
接收端看到坏帧
```

## Backpressure

AXI-Stream 通过 `tready` 施加反压。

```text
下游 ready = 0
  |
上游必须保持数据稳定
  |
不能丢 beat
```

如果上游不能停，例如某些实时流，就必须有足够 FIFO 或丢包策略。

## 完整运行时序

```text
Firmware 配置 clock/reset
  |
GT lock
  |
PCS block lock
  |
MAC enable
  |
DMA ring 初始化
  |
投递 RX buffer
  |
link up
  |
允许 TX/RX traffic
  |
RX 中断处理入包
  |
TX descriptor 发送出包
```
